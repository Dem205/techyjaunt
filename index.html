HTML Elements
You Should Not Be Using
In 2023

For my assignment, I'd be enlisting some HTML elements
that can be termed "old-fashioned" in this age and time.
These elements, though not depreciated, have their alternatives
and they are, by far, the most recommended.

The < b> Element

Some may disagree, but the < b > element has gradually lost its place in HTML, although not deprecated. The < b > element was used for bold text. However, CSS properties like font-weight achieves the same effect.

Read more

The < center > Element

The < center > element was used to wrap content
that you wanted to be horizontally centered.
CSS provides more flexible centering techniques using properties
like margin and text-align.

Read more

The < strike > Element

The < strike > element was built to strikethrough elements/words.
However, this same effect can be achieved
using the 'text-decoration' property in CSS.

Read more

The < u > Element

The < u > element was used for indicating underlined text.
In CSS, text-decoration: underline; can achieve the same effect.

Read more

The < table > Element for layout

If you still use < table > for layouts, please, get out of the cave,
I beg of you (;_;). CSS's display property (e.g., flex, grid) is now
the preferred method for creating layouts.
